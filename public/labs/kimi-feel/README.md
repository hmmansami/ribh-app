## Research Foundation

### One-Click UX Patterns Analyzed

| Pattern | Core Principle | Applied In |
|---------|---------------|------------|
| **Amazon 1-Click** | Pre-load everything; single button = instant transaction | v1: The "One Button" CTA with trust strip |
| **Uber Request** | Show outcome before action; live status after tap | v2: Revenue preview + animated timeline |
| **Shop Pay** | Express checkout promise; recognizable trust signals | Both: Security pills, "one click" microcopy |
| **Stripe Link** | Autofill magic; minimal fields | v1: Zero-input activation flow |

### Key Insights Applied

1. **Pre-Load Trust**: Both variants show "everything ready" before the CTA
2. **Outcome Preview**: Revenue projection visible before activation
3. **Post-Click Timeline**: Clear visualization of what happens after click
4. **Zero Friction**: No forms, no setup, just one action
5. **Control Assurance**: "You can pause anytime" messaging

## Variants

### v1-kimi-oneclick.html — "The Green Check"
**Philosophy:** Amazon 1-Click style absolute simplicity

- Single dominant green CTA with shimmer effect
- Three-dot trust strip (Secure, Instant, Control)
- Preview cards showing expected outcome
- Timeline that animates on click
- Success overlay with confirmation

**Best for:** Store owners who want maximum simplicity and immediate confirmation.

---

### v2-kimi-oneclick.html — "The Revenue Focus"
**Philosophy:** Uber/Shop Pay style outcome-first design

- Large revenue number as hero element
- Connection status dots (visual trust)
- Stacked preview cards with iconography
- Animated timeline with fill progression
- Live indicator in success state

**Best for:** Store owners motivated by clear financial outcomes.
